Custom CSS

For power users. The color pickers above cover most needs; this lets you go further by styling the two highlight selectors directly.

Selectors: ::highlight(better-find-all) and ::highlight(better-find-current). Rules here override the pickers above.

Reuse your chosen colors via variables: var(--bf-all-bg), var(--bf-all-text), var(--bf-current-bg), var(--bf-current-text).

Highlights use the browser's native CSS Custom Highlight API, which paints text ranges without touching the page. That keeps it fast and non-destructive, but the browser only honors a fixed set of properties on it:

✔ color, background-color, text-decoration (colored / wavy underlines, strikethrough), text-shadow (glow), -webkit-text-stroke (outlined text)

✖ border, border-radius, box-shadow, padding, animation / transition — silently ignored (they need a real element, which highlights are not).
